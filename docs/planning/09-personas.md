# 09-personas.md: User Personas

## MVP 캡슐 (요구사항 핵심 10줄)

> **목표**: 흩어진 추억을 안전하게 한 곳에 보존  
> **페르소나**: NAS+HDD+스마트폰에 20만+ 장 분산된 IT 개발자  
> **핵심 기능**: F1 중복감지(2-Stage), F2 베스탈 선별(AI추천+사용자확인), F4 내보내기(단순복사)  
> **North Star**: 중복 그룹 발견율 95%+ / 오탐율 5% 미만  
> **제약**: 20만 장 전체 스캔 30분 이내, 폴더별/기간별 선택 스캔, 증분 스캔  
> **위험**: 보정본 감지 정확도 (pHash 임계값)  
> **완화 전략**: Hamming Distance 임계값 A/B 테스트 (8 vs 10 vs 12)  
> **범위 외**: 메타데이터 통합(v2), 동영상(v2), 모바일(v2)  
> **Input Metric**: 스캔 완료율, 사용자 판정 완료율  
> **다음 단계**: `/screen-spec`으로 화면 명세 생성

|------|------|
| **이름** | "개발자 본인" (익명) |
| **나이** | 30~40세 |
| **직업** | 백엔드/풀스택 개발자 |

| **학력** | 학사 이상 (CS 또는 관련) |
| **수입** | 연 $60K~$120K (미국 기준) |
| **기술 수준** | 높음 (CLI, 디버깅 편함) |
| **거주지** | 도시 (대도시 선호) |

### 1.2 디지털 환경

**기기 구성**

```
Windows PC (주력)
├─ CPU: i7/i9 8코어+
├─ RAM: 16GB+
├─ SSD: 500GB (OS)
└─ 외장 HDD: 4~10TB

NAS (Synology/QNAP)
├─ 용량: 12~24TB
├─ 사진 저장: 20~50TB 축적
└─ 24/7 전원 ON

스마트폰 (백업용)
├─ iOS 또는 Android
├─ 매달 1~2GB 백업
└─ 클라우드 피함 (프라이버시)

외장 HDD (보관용)
├─ 용량: 2~4TB
├─ 보상분 + 정리분 보관
└─ 분기별 백업
```

### 1.3 사진 환경

**데이터 규모**

```
총 사진: 200,000장+
├─ 카메라: 100,000장 (Canon EOS R5)
├─ 스마트폰: 50,000장 (iPhone 14 Pro)
├─ 드론: 30,000장 (DJI Air 3)
└─ 여행: 20,000장 (기타)

저장 위치:
├─ NAS /Photos: 150,000장
├─ 외장 HDD: 50,000장
└─ 스마트폰: 5,000장 (동기화)

시간축:
├─ 2005~2015: 카메라 입문 (10,000장)
├─ 2015~2020: 적극 촬영 (100,000장)
└─ 2020~2024: 폭증 (90,000장) + 드론
```

### 1.4 고통점 (Pain Points)

**Problem 1: 파편화**
```
증상:
- 폰 → NAS 복사 (이름 규칙 다름)
- 카메라 → 외장HDD (어디있는지 모름)
- 드론 → 여러 폴더에 뿌려짐
- 백업본 여러 개 (어느 게 최신인지 불명)

영향:
- 같은 사진 여러 번 존재
- 원본/복사본 구분 불가
- 메타데이터 손실
- 관리 스트레스
```

**Problem 2: 중복 폭발**
```
증상:
- 백업 실패 후 재복사 (중복 발생)
- 보정본 원본 함께 저장
- 스마트폰 자동 백업 + 수동 복사 (중복)
- 시간이 지나 어느 게 중복인지 모름

수량:
- 추정 중복: 30,000~50,000장
- 절약 가능 용량: 50GB+
- 심리적 불안감: 높음
```

**Problem 3: EXIF 손실**
```
증상:
- 보정본은 원본 촬영시각 손실
- 압축 과정에서 메타데이터 제거
- 여러 폴더 병합 시 시각 정보 불일치

결과:
- 촬영 순서 재구성 불가
- 여행 타임라인 복원 어려움
- 감정적 연결고리 약화
```

**Problem 4: 제어권 부재**
```
현상:
- Google Photos: AI가 알아서 하는데, 기준이 뭔지 모름
- Apple Photos: 자동 선택 결과만 봄 (이의 제기 어려움)
- 수동 정리: 20만 장을 사람이 검토? 불가능

욕구:
- "AI가 추천해줄래? 그런데 내가 최종 판정할게"
- "어떻게 선택했는지 이유를 알고 싶어"
- "내 기준대로 커스터마이징하고 싶어"
```

**Problem 5: 신뢰성**
```
우려:
- 클라우드 서비스: 개인정보 유출 우려
- 해시 도구: 정확도 낮음 (80%대)
- 자동 도구: 잘못 지우면 복구 불가능

기대:
- "정말 중복을 95%+ 맞출 수 있을까?"
- "실수로 원본을 지우지는 않을까?"
- "NAS 네트워크 불안정해도 괜찮을까?"
```

**Problem 3: 신뢰 부족**
```
우려:
```
