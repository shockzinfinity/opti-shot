# 04-user-stories.md: User Stories & Acceptance Criteria

## MVP 캡슐 (요구사항 핵심 10줄)

> **목표**: 흩어진 추억을 안전하게 한 곳에 보존  
> **페르소나**: NAS+HDD+스마트폰에 20만+ 장 분산된 IT 개발자  
> **핵심 기능**: F1 중복감지(2-Stage), F2 베스탈 선별(AI추천+사용자확인), F4 내보내기(단순복사)  
> **North Star**: 중복 그룹 발견율 95%+ / 오탐율 5% 미만  
> **제약**: 20만 장 전체 스캔 30분 이내, 폴더별/기간별 선택 스캔, 증분 스캔  
> **위험**: 보정본 감지 정확도 (pHash 임계값)  
> **완화 전략**: Hamming Distance 임계값 A/B 테스트 (8 vs 10 vs 12)  
> **범위 외**: 메타데이터 통합(v2), 동영상(v2), 모바일(v2)  
> **Input Metric**: 스캔 완료율, 사용자 판정 완료율  
> **다음 단계**: `/screen-spec`으로 화면 명세 생성

## US1: 20만 장 중복 감지

**As a** IT 개발자  
**I want to** NAS+HDD에 흩어진 20만+ 장 사진에서 중복을 자동으로 찾고  
**So that** 어떤 파일이 중복인지 한눈에 알 수 있고, 정리 전략을 세울 수 있다

### 1.3.1 Acceptance Criteria

```
[ ] 1. 스캔 범위 지정
      - 다중 폴더 선택 가능 (C:\, \\NAS\shared 등)
      - 재귀 스캔 (하위 폴더 자동 포함)
      
[ ] 2. 감지 정확도
      - 중복 발견율 95%+ (벤치마크: 100장 테스트셋, 실제 중복 95개 이상)
      - 오탐율 5% 미만 (거짓 양성 < 5개)
      - 보정본 감지 포함 (원본 + 편집본이 같은 그룹)
      
[ ] 3. 성능
      - 20만 장 전체 스캔: 30분 이내
      - 스캔 중 UI 반응성: < 100ms
      
[ ] 4. 결과 표시
      - 그룹별 파일 목록 표시
      - 예상 절약 용량 계산 (베스탈 제외한 중복분)
      - 각 파일의 해시, EXIF, 파일 크기 표시
```

### 1.3.2 구현 태스크

- **T1.1**: pHash 계산 구현 (sharp)
- **T1.2**: BK-Tree 인덱싱 & 쿼리 구현
- **T1.3**: 2차 검증 (ORB + SSIM) 구현
- **T1.4**: 보정본 감지 알고리즘
- **T1.5**: 그룹화 로직
- **T1.6**: 테스트 & 정확도 검증

## US2: AI 추천 베스탈 선별

**As a** IT 개발자  
**I want to** 각 중복 그룹에서 AI가 최고 품질 이미지(베스탈)를 추천해주고  
**So that** 한눈에 어떤 파일을 보관할지 알 수 있고, 이의가 있으면 직접 수정할 수 있다

### 1.3.3 Acceptance Criteria

```
[ ] 1. 품질 점수 계산
      - Laplacian Variance (선명도)
      - Brightness (노출)
      - Contrast (색감)
      - 종합 점수: 0~100점
      
[ ] 2. 베스탈 추천
      - 그룹 내 상위 3개 순위 추천
      - 1순위를 기본 베스탈로 표시
      - 추천 근거 설명 (예: "선명도 95점, 노출 양호")
      
[ ] 3. 사용자 오버라이드
      - 베스탈 수동 변경 가능
      - AI 추천 무시하고 직접 선택 가능
      
[ ] 4. 사용자 만족도
      - 베스탈 추천 수동 수정율 < 20% (만족도 높음)
```

### 1.3.4 구현 태스크

- **T2.1**: 품질 점수 계산 (Laplacian, Brightness, Contrast)
- **T2.2**: 베스탈 추천 알고리즘
- **T2.3**: UI 상에서 수동 변경 기능
- **T2.4**: 테스트 & 사용자 만족도 검증

## US3: 실시간 스캔 진행 상황 확인

**As a** IT 개발자  
**I want to** 스캔 중 실시간으로 발견되는 중복 그룹을 볼 수 있고  
**So that** 스캔이 제대로 진행되는지 확인할 수 있고, 언제쯤 끝날지 예측할 수 있다

### 1.3.5 Acceptance Criteria

```
[ ] 1. 진행 상황 표시
      - 프로그레스바 (0~100%)
      - 처리된 파일 수 / 전체 파일 수
      - 경과 시간 / 예상 남은 시간
      - 발견된 그룹 수 / 예상 총 그룹 수
      
[ ] 2. 실시간 발견 피드
      - 새로 발견되는 중복 그룹을 하단 피드에 표시
      - "Group 102: 12개 파일 발견 (절약 25MB)" 형식
      
[ ] 3. 제어 버튼
      - Pause (일시정지 가능)
      - Cancel (중단 가능, 그 때까지의 결과는 보관)
      - Background (백그라운드로 축소)
      
[ ] 4. 복구 가능성
      - 스캔 중단 후 재개 시 이전 진행 상황부터 계속
```

### 1.3.6 구현 태스크

- **T3.1**: 스캔 상태 저장/복구 (SQLite)
- **T3.2**: 실시간 UI 업데이트 (React + Zustand)
- **T3.3**: 진행률 계산 알고리즘
- **T3.4**: Pause/Cancel/Background 제어

## US4: 베스탈을 원하는 폴더로 내보내기

**As a** IT 개발자  
**I want to** 선택된 베스탈들을 외장HDD나 정리 폴더로 복사하거나 이동할 수 있고  
**So that** 원본 보관 / 정리본 별도 보관으로 안전하게 정리할 수 있다

### 1.3.7 Acceptance Criteria

```
[ ] 1. 내보내기 액션
      - Copy (베스탈을 대상 폴더로 복사, 원본 유지)
      - Move (베스탈을 대상 폴더로 이동, 원본 삭제 → 휴지통)
      
[ ] 2. 대상 폴더 지정
      - 로컬 폴더 선택 가능
      - NAS 경로도 지원 (\\NAS\shared)
      - 부분 경로 지원 (예: D:\Curated)
      
[ ] 3. 안전성
      - Copy/Move 후 파일 무결성 검증 (체크섬)
      - 대상 파일 존재 시 처리 정책 (skip/rename/overwrite)
      - Move 실패 시 롤백 가능
      
[ ] 4. 결과 보고
      - 성공 / 실패 카운트
      - 전송된 총 용량
      - 실패한 파일 목록 (상세 에러)
      - 작업 로그 저장
```

### 1.3.8 구현 태스크

- **T4.1**: Copy 구현
- **T4.2**: Move 구현 (휴지통 연동)
- **T4.3**: 파일 무결성 검증 (체크섬)
- **T4.4**: 롤백 메커니즘
- **T4.5**: 결과 로깅

## US5: 실수로 삭제한 사진 30일 내 복원

**As a** IT 개발자  
**I want to** Delete를 누른 사진이 휴지통에 30일 보관되고  
**So that** 30일 이내 실수를 발견하면 복원할 수 있고, 30일 후 자동 삭제로 정리된다

### 1.3.9 Acceptance Criteria

```
[ ] 1. Soft Delete 메커니즘
      - 삭제 시 파일 즉시 제거 X
      - 휴지통 폴더로 이동 (.optishot/trash/)
      - 메타데이터 기록 (삭제 시각, 원본 경로)
      
[ ] 2. 휴지통 UI
      - 삭제된 사진 목록 표시
      - 각 사진의 삭제 시각 + 남은 보관 기간 표시
      - 복원 버튼으로 원본 경로로 복구
      
[ ] 3. 자동 정리
      - 30일 경과한 파일 자동 삭제
      - 스케줄러 또는 앱 시작 시 정리
      
[ ] 4. 신뢰성
      - 삭제 작업 로그 (감사 추적)
      - 복원 실패 시 알림
```

### 1.3.10 구현 태스크

- **T5.1**: Trash 폴더 관리
- **T5.2**: 삭제 시 휴지통으로 이동 구현
- **T5.3**: 휴지통 UI 화면
- **T5.4**: 자동 정리 스케줄러
- **T5.5**: 복원 기능

## US6: 폴더별/기간별 선택 스캔

**As a** IT 개발자  
**I want to** 전체 스캔뿐 아니라 특정 폴더나 날짜 범위만 스캔할 수 있고  
**So that** 빠르게 원하는 영역만 점검하고 이전 스캔 결과와 병합할 수 있다

### 1.3.11 Acceptance Criteria

```
[ ] 1. 스캔 모드
      - 전체 스캔 (모든 폴더)
      - 폴더별 스캔 (특정 폴더만)
      - 기간별 스캔 (2024-01-01~2024-12-31)
      - 증분 스캔 (마지막 스캔 이후 신규/변경 파일만)
      
[ ] 2. 기간 필터
      - EXIF 촬영 시각 기반
      - 날짜 피커로 선택
      
[ ] 3. 결과 병합
      - 이전 스캔 결과와 자동 병합
      - 새 그룹 추가, 기존 그룹 업데이트
      
[ ] 4. 성능
      - 폴더별 스캔 < 5분 (예: 1000장)
      - 증분 스캔 < 1분 (예: 신규 100장)
```

### 1.3.12 구현 태스크

- **T6.1**: 스캔 모드 선택 UI
- **T6.2**: 기간 필터 로직 (EXIF 기반)
- **T6.3**: 증분 스캔 쿼리 (mtime 비교)
- **T6.4**: 결과 병합 알고리즘

## US7: 설정으로 임계값 조정

**As a** IT 개발자  
**I want to** pHash 임계값, SSIM 임계값, Time-Window 등을 설정에서 조정할 수 있고  
**So that** 보정본 감지 정확도를 튜닝하고 내 요구사항에 맞출 수 있다

### 1.3.13 Acceptance Criteria

```
[ ] 1. 조정 가능한 파라미터
      - pHash Hamming Distance 임계값 (기본 8, 범위 4~16)
      - SSIM 임계값 (기본 0.85, 범위 0.5~0.95)
      - Time-Window (기본 1시간, 범위 0~24시간)
      - 병렬 스레드 수 (기본 8, 범위 2~16)
      - 배치 크기 (기본 100, 범위 10~500)
      
[ ] 2. 기본값 제공
      - 균형잡힌 기본값으로 즉시 사용 가능
      - 도움말: 각 파라미터의 영향 설명
      
[ ] 3. A/B 테스트 지원
      - 프리셋: "Conservative" (발견율↓, 오탐↓)
      - 프리셋: "Balanced" (기본)
      - 프리셋: "Sensitive" (발견율↑, 오탐↑)
      
[ ] 4. 설정 저장/로드
      - 설정은 SQLite에 저장
      - 앱 재시작 시 자동 복구
```

### 1.3.14 구현 태스크

- **T7.1**: 설정 UI 화면
- **T7.2**: 설정 저장/로드 (SQLite)
- **T7.3**: 파라미터 유효성 검사
- **T7.4**: 프리셋 제공

## US8: NAS에서 손쉽게 스캔 시작

**As a** NAS 관리자  
**I want to** NAS 폴더를 선택해서 한 번의 클릭으로 중복 감지를 시작하고  
**So that** 복잡한 설정 없이 쉽게 NAS를 정리할 수 있다

### 2.3.1 Acceptance Criteria

```
[ ] 1. NAS 폴더 추가
      - "폴더 추가" 버튼 클릭
      - \\NAS\shared 경로 직접 입력 또는 브라우저에서 선택
      - 여러 폴더 추가 가능
      
[ ] 2. 스캔 시작
      - 선택된 폴더 목록 표시
      - "스캔 시작" 버튼으로 원클릭 시작
      - 기본값으로 자동 설정 (임계값 등)
      
[ ] 3. 진행 상황 표시
      - 초보자도 이해하기 쉬운 프로그레스 표시
      - "3,450장 검사 중... 남은 시간 약 5분"
      
[ ] 4. 오류 처리
      - NAS 경로 없음 → 명확한 에러 메시지
      - 네트워크 연결 끊김 → 재시도 옵션
```

### 2.3.2 구현 태스크

- **T8.1**: 폴더 브라우저 (NAS 경로 지원)
- **T8.2**: 폴더 추가/제거 UI
- **T8.3**: 네트워크 오류 처리

## US9: 베스탈 자동 정렬하고 내보내기

**As a** NAS 관리자  
**I want to** 정렬된 베스탈들을 외장HDD로 한 번에 복사하고  
**So that** NAS 공간을 절약하고 원본은 백업, 정리본은 별도 보관할 수 있다

### 2.3.3 Acceptance Criteria

```
[ ] 1. 베스탈 자동 정렬
      - AI 추천 베스탈을 그룹별로 자동 정렬
      - 사용자 판정 필요 최소화
      
[ ] 2. 대상 폴더 지정
      - 외장HDD 드라이브 (D:\, E:\ 등) 선택
      - NAS 다른 영역도 가능
      
[ ] 3. 한 번에 복사
      - "모두 내보내기" 버튼
      - 진행 상황 실시간 표시
      - 완료 후 "완료: 50,000장 내보냄 (23GB)" 메시지
      
[ ] 4. 신뢰성
      - 복사 중 네트워크 끊김 → 복구 가능
      - 체크섬으로 파일 무결성 검증
```

### 2.3.4 구현 태스크

- **T9.1**: "모두 내보내기" 기능
- **T9.2**: 네트워크 안정성 (재시도)
- **T9.3**: 용량 계산 및 예상 시간

## 3. Common Story: 다중 실행 방지

**As a** 모든 사용자  
**I want to** OptiShot이 동시에 2개 이상 실행되지 않도록 방지되고  
**So that** 데이터베이스 충돌이나 스캔 중복을 피할 수 있다

### 3.1 Acceptance Criteria

```
[ ] 1. 단일 인스턴스 강제
      - 앱 시작 시 이미 실행 중이면 기존 창 활성화
      - 새로운 창 띄우지 않음
      
[ ] 2. 뮤텍스 사용
      - Named Mutex로 인스턴스 제어
      - Named: "OptiShot-SingleInstance"
      
[ ] 3. 우아한 처리
      - 사용자에게 알리지 않음 (조용히 처리)
      - 기존 창이 최소화 상태면 복구
```

### 3.2 구현 태스크

- **T10.1**: Named Mutex 구현
- **T10.2**: 기존 창 활성화 로직
