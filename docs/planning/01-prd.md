# 01-prd.md: Product Requirements Document

## MVP 캡슐 (요구사항 핵심 10줄)

> **목표**: 흩어진 추억을 안전하게 한 곳에 보존  
> **페르소나**: NAS+HDD+스마트폰에 20만+ 장 분산된 IT 개발자  
> **핵심 기능**: F1 중복감지(2-Stage), F2 베스탈 선별(AI추천+사용자확인), F4 내보내기(단순복사)  
> **North Star**: 중복 그룹 발견율 95%+ / 오탐율 5% 미만  
> **제약**: 20만 장 전체 스캔 30분 이내, 폴더별/기간별 선택 스캔, 증분 스캔  
> **위험**: 보정본 감지 정확도 (pHash 임계값)  
> **완화 전략**: Hamming Distance 임계값 A/B 테스트 (8 vs 10 vs 12)  
> **범위 외**: 메타데이터 통합(v2), 동영상(v2), 모바일(v2)  
> **Input Metric**: 스캔 완료율, 사용자 판정 완료율  
> **다음 단계**: `/screen-spec`으로 화면 명세 생성

## 1. 제품 개요

### 1.1 비전 & 가치 제안

**OptiShot**: 내 사진을 내 규칙으로, 단계별로 확인하며 정리하는 로컬 데스크톱 도구

**Cogito**: 수십 년간 흩어진 추억을 안전하게 한 곳에 보존

**주요 가치**
- **로컬 처리**: 100% 로컬 처리, 클라우드 전송 없음 (개인정보 보호)
- **제어권**: AI 추천이지만 최종 판정은 사용자 (신뢰)
- **정밀도**: 2-Stage Detection (pHash + ORB/SSIM)으로 95%+ 발견율, 5% 이하 오탐
- **성능**: 20만 장 전체 스캔 30분 이내
- **안전성**: Soft Delete (휴지통 30일 보관), 원본 절대 안 건드림

### 1.2 타겟 사용자

**Primary Persona**: IT 개발자 (5년+ 경력)
- Windows PC + NAS + 외장HDD + 스마트폰 멀티 환경
- 20만+ 장 / 20년+ 축적 사진
- 파편화, 중복 폭발, EXIF 손실에 고통

**Secondary Persona**: NAS 사진 관리 고민러
- IT 친화적, NAS 소유
- 5만~50만+ 장
- 쓸만한 도구 없어서 답답함

## 2. 기능 명세

### 2.1 MVP 기능 (우선순위 P0~P2)

#### F1: 유사/중복 감지·그룹화 (P0 - 핵심)

**설명**: pHash + Time-Window로 1차 후보 감지, ORB/SSIM으로 2차 정밀 검증. BK-Tree 인덱싱. 보정본도 같은 그룹으로 묶음

**입력**
- 스캔 대상 폴더 (단일 또는 다중)
- 스캔 모드 (전체/폴더별/기간별/증분)
- pHash 임계값 (기본값 8, 튜닝 가능)
- SSIM 임계값 (기본값 0.85)

**출력**
- 중복 그룹 목록 (Group ID, 파일 목록, 그룹 크기, 예상 절약 용량)
- 각 파일별 해시, EXIF, 파일 정보
- 보정본 마크

**알고리즘**
1. 1차: pHash 계산 → BK-Tree로 Hamming Distance ≤ 임계값 후보 추출
2. 2차: 후보 쌍에 대해 ORB (특징점) + SSIM (구조 유사도) 검증
3. Time-Window (기본 1시간): 촬영 시각 가까운 파일끼리만 매칭 (오탐 감소)
4. 보정본 감지: Metadata 및 파일명 패턴 (+원본 파일 존재 확인)

**엣지 케이스**
- EXIF 없는 사진 → 콘텐츠 기반만 사용 (Time-Window 사용 안 함)
- 보정본+원본 → 같은 유사 그룹으로 묶음 (pHash 임계값 관대하게, 기본 8)
- 파일명 무의미 → 파일명 의존 로직 없음
- 동영상 → MVP 제외

**완료 조건**
- 중복 발견율 95%+ (벤치마크: 100장 테스트셋)
- 오탐율 5% 미만
- 20만 장 스캔 30분 이내

---

#### F2: 품질 평가·베스탈 선별 (P1 - 우선도 높음)

**설명**: Laplacian Variance 기반 품질 점수. AI가 베스탈 추천, 사용자가 확인/수정

**입력**
- 감지된 중복 그룹
- 각 파일의 해시, EXIF, 이미지 메타데이터

**출력**
- 각 파일별 품질 점수 (0~100)
- 그룹별 베스탈 추천 (상위 N개 이미지 + 점수 근거)
- 사용자가 베스탈 확인/수정 후 최종 판정

**품질 메트릭**
- **Laplacian Variance**: 초점 명확도 (값 클수록 좋음)
- **Brightness**: 노출 적절도
- **Contrast**: 색감 정보량
- **Sharpness**: 모션 블러 감지

**AI 추천 로직**
- 그룹 내 모든 파일 점수 계산
- 상위 3개 순위 추천
- 추천 근거 표시 (품질 점수, 선명도 점수 등)
- 사용자가 우선순위 변경 가능

**완료 조건**
- 품질 점수 계산 오류 최소화 (벤치마크: 50장 테스트)
- 베스탈 추천 수동 수정율 < 20% (사용자 만족도)

---

#### F4: 내보내기 (Export) (P2 - MVP 포함)

**설명**: MVP: 선택한 베스탈을 지정 폴더로 복사/이동. v2: 규칙 기반 자동 내보내기

**입력 (MVP)**
- 선택된 베스탈 파일 목록
- 대상 폴더 경로
- 액션: Copy 또는 Move

**출력 (MVP)**
- 내보내기 완료 로그 (성공/실패 카운트)
- 원본 파일 상태 (Move 시 삭제, Copy 시 유지)

**액션**
- **Copy**: 베스탈을 대상 폴더로 복사 (원본 유지)
- **Move**: 베스탈을 대상 폴더로 이동 (원본 삭제 → 휴지통에 저장)

**완료 조건**
- 베스탈 내보내기 100% 성공
- 파일 무결성 확인 (체크섬 비교)

#### F3: 메타데이터 통합/복원

**설명**: MetadataExtractor로 EXIF/XMP 읽기, 손실된 메타데이터 복원/병합

## 3. 비기능 요구사항

### 3.1 성능 (Performance)

| 항목 | 요구사항 |
|------|--------|
| 20만 장 전체 스캔 (pHash 1차) | 30분 이내 |
| 스캔 중 UI 반응성 | < 100ms |
| 메모리 사용량 | 4GB 이내 |
| 배치 처리 크기 | 100개 이미지/배치 |

### 3.2 스캔 모드 (Scan Modes)

- **전체 스캔**: 지정 폴더 + 하위 전체
- **폴더별 스캔**: 특정 폴더만 선택적 스캔
- **기간별 스캔**: 지정 날짜 범위만 스캔 (EXIF 기반)
- **증분 스캔**: 마지막 스캔 이후 신규/변경 파일만

### 3.3 안전성 (Safety)

- **Soft Delete**: 삭제된 파일은 휴지통(폴더)에 30일 저장 후 자동 삭제
- **원본 보호**: 사용자 명시적 선택 없이 원본 절대 변경 안 함
- **작업 로그**: 모든 삭제/이동/복사 작업 기록
- **복구 지원**: 30일 내 휴지통에서 복원 가능

### 3.4 개인정보 보호 (Privacy)

- **100% 로컬 처리**: 클라우드 전송 없음
- **데이터베이스 로컬 저장**: SQLite (프로젝트 폴더 내)
- **이미지 캐시 로컬**: 스캔 결과 해시/EXIF만 저장
- **권한 최소화**: 사용자 선택 폴더만 접근

### 3.5 호환성 (Compatibility)

- **파일 포맷**: JPEG, PNG, WEBP, HEIC (기본)
- **플랫폼**: macOS / Windows / Linux (Electron + Node.js)
- **NAS 경로**: SMB/NFS (공유 폴더 지원)
- **외장 드라이브**: USB 3.0+ (NTFS, exFAT, HFS+)

## 7. 위험 & 완화 전략

### 7.1 Top Risk: 보정본 감지 정확도

**위험**: pHash 임계값 설정에 따라 발견율과 오탐율이 극단적으로 달라질 수 있음

**영향도**: 높음 (발견율 80~99% 편차)

**완화 전략**
1. **A/B 테스트**: Hamming Distance 임계값 (8, 10, 12) 테스트
   - 8: 민감도 높음 (발견율↑, 오탐율↑)
   - 10: 균형 (목표)
   - 12: 보수적 (발견율↓, 오탐율↓)
2. **벤치마크 데이터셋**: 100장 (중복 20개, 보정본 10개) 구성
3. **사용자 피드백 루프**: 초기 사용자에게 피드백 수집, 임계값 조정
4. **기본값 보수적 설정**: 초기 기본값 8 (낮은 오탐) → 점진적 개선

### 7.2 Secondary Risk: 성능 저하

**위험**: 20만+ 장 스캔 시 pHash 계산/비교에서 병목

**완화 전략**
1. **병렬 처리**: 배치 100개 이미지/스레드 (8 스레드)
2. **인덱싱**: BK-Tree로 비교 대상 1% 이하로 필터링
3. **점진적 UI 업데이트**: 배치 완료마다 진행 상황 갱신 (UI 프리징 방지)
